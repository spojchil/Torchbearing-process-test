# 团队行为准则

Torchbearing 希望维持一个专业、尊重事实、能够安全讨论分歧的协作环境。本准则适用于 Issue、Pull Request、Review、会议和其他项目沟通渠道。

## 鼓励的行为

- 讨论方案、代码和证据，不攻击个人；
- 反对一个方案时说明事实、风险、备选方案和验证方法；
- 尊重不同经验水平，为不熟悉 GitHub 或技术栈的同事提供可操作帮助；
- 及时回应 Review，公开记录影响项目的关键结论；
- 主动承认不确定性和错误，通过讨论、Spike 或测试收敛；
- 保持 Issue 和 PR 范围清楚，尊重他人的评审时间；
- 对仓库内容、凭据和内部信息承担保密责任。

## 不可接受的行为

- 人身攻击、嘲讽、歧视、骚扰或威胁；
- 未经沟通覆盖、删除或强推他人的工作；
- 绕过 Issue、Review 或指定 Merger 直接修改正式仓库；
- 隐瞒关键风险、伪造验证结果或将无法解释的 AI 输出冒充完成成果；
- 在公开位置传播凭据、内部数据或安全漏洞细节；
- 利用仓库权限进行与项目无关或未经授权的操作。

## 分歧处理

技术和产品分歧优先在对应 Draft PR、Issue 或代码 PR 中处理：

1. 明确共同目标；
2. 分别列出事实、假设、风险和备选方案；
3. 能验证的争议通过 Spike、原型或测试验证；
4. 无法收敛时，请仓库协作者、仓库创建者、导师或助教作出判断并记录理由。

决定作出后可以保留不同意见，但不得绕开已经记录的结论。

## 报告与执行

如遇不当行为，请通过团队私密渠道联系仓库创建者或仓库协作者。处理时保护报告人隐私，并根据影响采取提醒、限制参与、撤销权限或上报组织管理者等措施。

仓库创建者负责本准则的最终执行；涉及仓库创建者本人的问题，交由组织管理者、导师或助教处理。
